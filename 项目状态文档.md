## 一、项目是什么

**目标:** 在云手机上伪装成真机(Samsung Galaxy S10),绕过腾讯 TerSafe/ACE 反作弊系统的全部检测层,让三角洲行动(com.tencent.tmgp.dfm)不封号运行。

**核心原理:** 五层纵深防御对抗 TerSafe 的六层检测体系。

### v5.6 → v5.8 演进

**v5.6 root cause (已修复):**
BPF 死代码被内核拒绝 → `Seccomp: 0` → game dies。

**v5.7 修复:**
- BPF v3: 线性链, tgkill→tkill→kill→file→ALLOW (31条指令)
- hook_log() 改写到 `/data/local/tmp/forge_hook.log`
- kill()/tgkill() libc hook 只拦 SIGKILL(9)/SIGTERM(15)

- 守护进程从 5s 降到 1s

**v5.8 新增 (P0/P1 升级):**
- **exit_group hook** — 检测返回地址,仅拦来自 libtersafe.so 的调用
- **getenv hook** — 拦截 LD_PRELOAD / LD_LIBRARY_PATH / ANDROID_ROOT 探测
- **dl_iterate_phdr hook** — 过滤 libforgehook / libtdmqimei_real 条目

- **dlsym hook** — 初始化阶段安全放行,防止死锁
- **dladdr hook** — 地址→名称查询返回 "libc.so" 替代真实库名
- **r_debug link_map 摘除** — 从 linker 模块链表删除 libforgehook 条目 (constructor 101)
- **opendir/readdir/readdir64** — 24 个检测工具名过滤
- **getaddrinfo hook** — 拦截 14 个 AC 域名 DNS 解析
- **connect hook** — 拦截 5 个 AC IP 段连接 (DNS 封锁的兜底)
- **GPU 伪装 (inline branch patch)** — glGetString → Adreno 740, eglQueryString → Qualcomm
- **BPF v4** — 移除文件 TRAP (v3 的 TRAP→SIGSYS 返回 -ENOSYS 会破坏游戏文件访问)
- **/proc/self/maps 动态过滤** — 运行时读真实 maps,strip 掉注入库行后返回 memfd
- **全量守护精灵** — 1s 周期,分层验证 (kill链每周期/all代码每3周期/BSS每5周期/UE4每2周期)

## 三、五层防护 vs 六层检测

```
TerSafe 检测层                     DeltaForge 对抗层
─────────────────────────────────────────────────────────
L1 网络层(IP/ASN)          →     getaddrinfo + connect 双 hook (14域名+5IP段), iptables 仅清旧规则
L2 系统层(属性/cpuinfo/GPU) →     libc hook 伪造 30+ 文件路径 + GPU inline branch patch
L3 Java层(Build/Properties) →     JNI_OnLoad 覆写 16 字段 + SystemProperties 4 原生方法 + __system_property_get 70+ 属性
L4 内核层(/proc/sys/virtio) →     47+ 路径隐藏 + seccomp-bpf v4 (信号拦截,无文件TRAP)
L5 运行时层(检测+kill)      →     119处代码补丁 + kill链全节点覆盖 + exit_group 地址检测
L6 行为层(跑刀模式)          →     (bot_runner.py,P3 待实现)

我们的五层:
  L1 seccomp-bpf v4 (constructor 49,最早,信号only)
  L2 maps 自隐藏 + r_debug 摘除 (constructor 50+101)
  L3 libc PLT hooks 30+ 函数 (so加载瞬间)
  L4 运行时代码补丁 (constructor 150,轮询等待 libtersafe.so)
  L5 外部 /proc/PID/mem 补丁 + 1s 分层回读守护
```

## 四、技术细节

### 4.1 BPF v4 — 信号拦截 (23条指令,零死代码,零文件TRAP)

```
arch(aarch64) → tgkill(131, sig1-31) → tkill(130, sig1-31) → kill(129, sig9/15) → ALLOW
                  ↓BLOCK                  ↓BLOCK                 ↓BLOCK(9,15 only)
```

v3 bug: 文件 syscall TRAP→SIGSYS 返回 -ENOSYS 对任何不在 FAKE_FILES/HIDDEN 中的路径都会失败,破坏所有游戏文件访问。
v4 fix: 文件拦截完全从 BPF 移除,改由 libc hooks (open/fopen/stat 等) 在用户态处理。TerSafe 用直接 syscall 读文件可以读到真实内容,但 kill 链阻断和网络封锁阻止任何发现被上报。

### 4.3 Library Hijack (最关键的部署步骤)

Android 8+ linker namespace 隔离,从 `/data/local/tmp/` dlopen 会返回 NULL。hijack 方式解决:

```sh
cd /data/app/.../com.tencent.tmgp.dfm.../lib/arm64/
mv libtdmqimei.so libtdmqimei_real.so    # 备份原版
cp libforgehook.so libtdmqimei.so        # 我们的 so 冒名顶替
```

游戏加载 `libtdmqimei.so` → 实际加载我们的 `libforgehook.so` → constructor 执行所有 hook 和补丁 → chainload 调用原版 `libtdmqimei_real.so`。

## 六、遗留风险

1. **GPU 指纹矛盾:** 云手机物理 Mali GPU,属性伪装 Adreno 740。glGetString/eglQueryString 已 inline branch patch,但服务端可能通过其他 GL 扩展查询检测。

2. **TerSafe 完整性校验:** 119 处补丁可能被 TerSafe 自身的 CRC 校验检测到。1s 分层回读+自动重打可以对抗被动还原,但无法对抗校验触发后的备用 kill 路径。

3. **kill(129) 只拦 SIGKILL/SIGTERM:** Android runtime 依赖 kill() 做进程组信令,拦截全部信号会导致瞬间崩溃。TerSafe 可能改用其他致命信号绕过。

4. **服务端行为检测:** 跑刀模式(固定路线/停留时间)会被服务端识别为非人类行为。bot_runner.py 为 P3 待实现。
