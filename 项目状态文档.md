## 三、工作流

### PC 端修改并推送

```sh
cd "D:\下载\cc-chain-forge (3)\DeltaForge-repo"
git add -A
git commit -m "描述"
git push origin master
```

### 云手机拉取、编译、部署（一条命令）

```sh
cd ~/DeltaForge && git pull origin master && sh cloud-agent/deploy.sh build
```

`deploy.sh build` 自动完成：
1. Termux clang 编译（`-pie`，非 `-static`）
2. 写临时子脚本绕过终端折行问题
3. `su` root 身份复制到 `/data/local/tmp/`
4. 设置权限并打印版本确认

### 启动游戏

```sh
su -c '/data/local/tmp/forge -l'
```

### 启动监控（另开 Termux 会话）

```sh
su -c '/data/local/tmp/forge_monitor -v > /data/local/tmp/forge_monitor.log 2>&1 &'
```

## 四、二进制文件

| 文件 | 位置（云机） | 用途 |
|------|------------|------|
| `forge` | `/data/local/tmp/forge` | 主控：内存补丁 + 文件清理 + 属性伪装 |
| `libforgehook.so` | `/data/local/tmp/libforgehook.so` | 注入游戏进程的 hook 库 |
| `forge_monitor` | `/data/local/tmp/forge_monitor` | 反作弊行为监控工具 |
| `injector` | `/data/local/tmp/injector` | ptrace 注入器 |
| `touch_injector` | `/data/local/tmp/touch_injector` | /dev/uinput 触摸注入 |

编译命令（在 `~/DeltaForge/cloud-agent/native/`）：
```sh
clang -pie -Os -Wall forge.c -o forge
clang -shared -fPIC -Os -Wall libforgehook.c -o libforgehook.so -ldl
clang -pie -Os -Wall forge_monitor.c -o forge_monitor
clang -pie -Os -Wall injector.c -o injector -ldl
clang -pie -Os -Wall touch_injector.c -o touch_injector
```

## 五、已实现的功能（v5.5）

### forge.c

- 内存补丁：libtersafe.so 61处代码 + 40处 BSS + libUE4.so 6处（共 107处）
- 文件清理：5秒循环守护，清 ano_tmp / tdm_tmp / qm 等目录
- 属性伪装：与 beyond1q fingerprint 一致（Samsung SM-G9730）
- iptables：仅清理旧残留规则，不做网络拦截

### libforgehook.so

- open/openat/fopen：虚假文件路由 + 路径隐藏
- getenv：拦截 LD_PRELOAD 检测
- dl_iterate_phdr：过滤自身被库枚举发现
- opendir/readdir：目录级隐藏
- `__system_property_get`：属性值拦截
- JNI `android.os.Build.*`：Java 层设备信息覆写
- seccomp-BPF：内核 syscall 层拦截
- /proc/self/status 伪造（TracerPid: 0）
- /proc/net/tcp 伪造（空连接表）
- null_redirect：纯上报文件重定向到匿名 memfd

### forge_monitor

- inotify 监听游戏数据目录
- /proc/[pid]/fd 枚举已打开文件
- /proc/[pid]/net/tcp 检测上报连接
- 输出 forge_monitor.log + 读取 forge_audit.log 覆盖缺口

## 六、当前项目状态

| 项目 | 状态 |
|------|------|
| 内存补丁 107/107 | ✅ 每次成功 |
| ptrace 注入 libforgehook.so | ✅ 成功（handle 非0） |
| 游戏进程启动 | ✅ |
| 游戏完成初始化 / 联网 | ❌ 约 7 秒后反作弊主动 SI_TKILL |

### 已确认的问题

**游戏启动 7秒后自杀（SI_TKILL）**

- `signal 11 (SIGSEGV), code -6 (SI_TKILL)` — 反作弊调用 `tgkill()` 主动发送，非真实内存错误
- 本次 session 修复了属性不一致（OPPO/Xiaomi → Samsung S10），但游戏仍被检测

**待排查的剩余检测点：**
1. libtersafe.so 代码被补丁后，其自身完整性校验（CRC）检测到修改
2. ptrace 注入时机在 TerSafe 初始化之后，早期检测已运行
3. 云手机硬件特征（GPU/传感器）无法通过属性 hook 覆盖
4. TerSafe 可能通过 Binder 或直接 syscall 读属性，绕过 `__system_property_get` hook

## 八、Termux 环境已知限制

| 限制 | 原因 | 解决方式 |
|------|------|---------|
| `-static` 编译失败 | Termux 无静态 libc | 改用 `-pie` |
| `resetprop` 找不到 | Magisk 路径非标准 | libforgehook hook 层兜底 |
| 长命令终端折行 | `su -c '...'` 换行被拆 | 写脚本文件再执行 |
| `python` 不存在 | 云机无 Python | forge_controller.py 从 PC 端运行 |
| forge 无法覆盖（Text file busy） | 旧 forge 正在运行 | deploy.sh 已加 pkill forge |
