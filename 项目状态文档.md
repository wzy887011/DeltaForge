## 一、项目是什么

**目标:** 在云手机上伪装成真机(Samsung Galaxy S10),绕过腾讯 TerSafe/ACE 反作弊系统的全部检测层,让三角洲行动(com.tencent.tmgp.dfm)不封号运行。

**核心原理:** 五层纵深防御对抗 TerSafe 的六层检测体系。

## 三、五层防护 vs 六层检测

```
TerSafe 检测层                     DeltaForge 对抗层
─────────────────────────────────────────────────────────
L1 网络层(IP/ASN)          →     (暂无,iptables 仅清旧规则)
L2 系统层(属性/cpuinfo/GPU) →     libc hook 伪造 30+ 文件路径
L3 Java层(Build/Properties) →     JNI_OnLoad 覆写 + __system_property_get hook
L4 内核层(/proc/sys/virtio) →     35+ 路径隐藏 + seccomp TRAP
L5 运行时层(检测+kill)      →     113处代码补丁 + seccomp 拦截 kill syscall
L6 行为层(跑刀模式)          →     (bot_runner.py,P3 待实现)

我们的五层:
  L1 seccomp-bpf (constructor 49,最早)
  L2 maps 自隐藏 (constructor 50)
  L3 libc PLT hooks (so加载瞬间)
  L4 运行时代码补丁 (constructor 150,轮询等待 libtersafe.so)
  L5 外部 /proc/PID/mem 补丁 + 5秒回读
```

### 4.2 Library Hijack (最关键的部署步骤)

Android 8+ linker namespace 隔离,从 `/data/local/tmp/` dlopen 会返回 NULL。hijack 方式解决:

```sh
# df-hijack-root.sh 做的事:
cd /data/app/.../com.tencent.tmgp.dfm.../lib/arm64/
mv libtdmqimei.so libtdmqimei_real.so    # 备份原版
cp libforgehook.so libtdmqimei.so        # 我们的 so 冒名顶替
```

游戏加载 `libtdmqimei.so` → 实际加载我们的 `libforgehook.so` → constructor 执行所有 hook 和补丁 → chainload 调用原版 `libtdmqimei_real.so`。

**deploy.sh 已自动处理:** 编译后自动 `find /data/app -name libtdmqimei_real.so` 找到 hijack 目录,用新版 so 覆盖。

### 4.4 终端操作注意事项

1. **`su -c "..."` 长路径折行截断:** Android shell 的 `su -c` 遇到换行符会截断命令。长命令必须先写成脚本文件再 `su -c 'sh script.sh'`。

2. **`/data/local/tmp/` 权限:** 目录属主 root,Termux 普通用户(u0_a73)无法覆盖已有文件。deploy.sh 必须通过 heredoc 生成子脚本,再用 `su -c` 执行。

3. **`/data/app/` 不可读:** 普通用户无法访问,查找 hijack 路径必须在 root 下进行。

4. **Termux 编译限制:** `-static` 不可用(无静态 libc)。`resetprop` 不存在(Magisk 未装)。

5. **MD5 交叉验证是唯一可信的部署确认:** deploy.sh 输出本地和远端 MD5,必须一致。

## 六、遗留风险

1. **GPU 指纹矛盾:** 云手机物理 Mali GPU,属性伪装 Adreno 740。`glGetString(GL_RENDERER)` 无法 hook,服务端可能检测到 GPU 不一致导致封号(非崩溃)。

2. **TerSafe 完整性校验:** 113 处补丁可能被 TerSafe 自身的 CRC 校验检测到。5秒回读+自动重打可以对抗被动还原,但无法对抗校验触发后的备用 kill 路径。

3. **`kill(129)` 无法在 seccomp 层拦截:** Android runtime 依赖 `kill()` 做进程组信令,拦截会导致瞬间崩溃。TerSafe 可能改用 `kill()` 绕过我们的 tgkill/tkill 拦截。

4. **服务端行为检测:** 跑刀模式(固定路线/停留时间)会被服务端识别为非人类行为。
